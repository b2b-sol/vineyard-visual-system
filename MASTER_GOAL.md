# Master Goal

Create and publish a complete, professional array of visual materials for constructing a vineyard information system. The project is produced by Codex and its subagents under one human owner, with the supplied workflow-discovery report as the foundational operational source of truth.

## Target repository

- Organization: `b2b-sol`
- Repository: `vineyard-visual-system`
- Visibility: public
- Long-running operating identity: `tiklebot`
- Primary branch: `main`

## Final product

The finished repository must contain a working browser-based **Vineyard Visual Atlas** and the structured source material behind it. A reviewer opening the final GitHub Pages site or running the repository locally must be able to inspect:

1. How a commercial vineyard operates across roles, seasons, organizations, records, decisions, and exceptions.
2. How a future vineyard information system should support those workflows.
3. Complete mobile and desktop interaction designs for the priority workflows.
4. A coherent, distinctive high-fidelity visual language and reusable design system.
5. Realistic connected synthetic operating data used by all visuals.
6. Complete state coverage, including blocked, delayed, offline, correction, conflict, and historical states.
7. Construction packets that another implementation agent could use without rediscovering the product.
8. A traceability chain from source evidence to workflow, scenario, product requirement, screen, component, and construction packet.
9. A Figma-ready export and migration bundle for future paid Figma/MCP/Code Connect work.
10. Evidence that the finished library passed autonomous workflow, UX, visual, accessibility, data, and completion reviews.

## Autonomy requirement

After the owner-to-bot authentication checkpoint, complete this project without routine human approvals. Use creator–critic–integrator loops, independent extraction, browser validation, automated checks, and a final Sol completion audit.

## Source-of-truth rule

The workflow report describes a coordination system, not merely a set of viticultural topics. Design around the recurring loops it identifies: observation to action, plan to crew assignment, application to compliance record, crop estimate to winery forecast, maturity updates to pick decision, harvest to weigh-ticket reconciliation, and field activity to payroll and cost reporting.

## Completion rule

Do not declare completion while any requirement in `DEFINITION_OF_DONE.md` is unmet. A scaffold, representative sample, attractive homepage, partial design system, or small set of screens is not completion.
