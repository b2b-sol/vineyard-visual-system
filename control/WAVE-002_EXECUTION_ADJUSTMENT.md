# Deliverable-first execution adjustment

Effective after the accepted WAVE-001 checkpoint, the project moves from infrastructure expansion to human-visible workflow delivery.

## Frozen foundation

The current workflow ontology, synthetic season, product contracts, permissions, synchronization model, schemas, validators, trace graph, wave orchestration, token system, CI, capture pipeline, and Figma mapping machinery are the default foundation for WAVE-002 through WAVE-005. Reuse them without extending their surface.

A foundation change is permitted only for a specific observed failure that existing checks cannot detect or prevent. The proposal must first record the uncovered failure, why current checks cannot cover it, and the expected effect on the final human-visible deliverable.

## Real progress

Progress is now measured primarily by accepted rendered screens and complete task walkthroughs: coherent normal-to-exception-to-recovery narratives; realistic desktop and mobile responsibility; visible actor, scope, state, timing, evidence, and connectivity context; an enabled next action inside the initial field viewport; meaningful exception and recovery states; composition variety; cross-wave visual consistency; and complete workflow coverage in the final presentation.

Assertion totals, graph size, contract counts, and generated metadata remain health signals, not delivery goals.

## Later consolidation

Final integration will consolidate or retire duplicate screen registries, per-wave generated mappings that have served their review purpose, the standalone factory walking slice and superseded planning material, repeated capture metadata, and other intermediates that no longer support an active build, review, migration, or audit need. Stable identifiers and editable canonical sources will be preserved.

## Remaining sequence

The vertical wave order remains the shortest coherent route because later harvest, settlement, grower, and audit work depends on the operating context established by earlier field decisions. Each remaining wave will start from its key user task and rendered workflow strip, compose high-fidelity desktop and field screens from the frozen system, walk normal and non-ideal paths with connected data, resolve visual review findings, and only then package construction and Figma handoff evidence.

The remaining priority order is workflow coherence, task realism, desktop/mobile composition, visible operating context, initial-viewport actionability, exception and recovery quality, template diversity, cross-wave consistency, and final consolidation and presentation.
