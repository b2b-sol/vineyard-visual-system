# Source

`Vineyard-Workflow-Discovery-report.md` is the foundational operational source. Preserve it unchanged as an evidence document. `evidence.json` normalizes its claims into stable IDs, structural source anchors, confidence classifications, limitations, workflow coverage, and field-validation questions. `bibliography.json` replaces opaque browsing handles with unique public URLs from primary or official industry publishers while retaining the original handles for provenance.

The report itself acknowledges stronger confidence in formal requirements and weaker confidence in informal day-to-day behavior. Carry that distinction into the product.

Run `npm run validate:evidence` to check schemas, report anchors, opaque provenance handles, public-source references, uniqueness, and coverage of all ten canonical workflow families. Network freshness is checked separately so deterministic builds never depend on third-party availability.
