# Atlas Implementation Instructions

- The browser output is the primary review surface.
- Keep routes stable and data-driven.
- Use reusable components and design tokens.
- Render meaningful loading, empty, offline, stale, blocked, conflict, correction, and historical states.
- Use Playwright for actual browser validation and screenshots.
- Preserve keyboard, responsive, print, and accessibility behavior.
- Avoid hard-coded one-off screen markup when a reusable data/component model fits.
