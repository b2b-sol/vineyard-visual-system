# Assumptions and Risks

## 1. Workflow report completeness

The report is strongest for formal requirements, public procedures, certification, and role expectations. Informal behavior—texts, whiteboards, verbal closure, contradictory numbers, and after-the-fact correction—is less certain. Preserve confidence and field-validation flags.

## 2. Product boundary

“Vineyard information system” could expand into accounting, winery production, CRM, payroll, and full ERP. This packet defaults to deep vineyard operations with explicit integration boundaries for adjacent systems.

## 3. Code-first visuals

The plan assumes React/SVG/Mermaid/token sources are acceptable canonical design artifacts. Figma remains a future destination. The Figma bundle mitigates migration risk.

## 4. Multi-agent editability

Independent revision is reliable only when artifacts are structured and shared. Flat images are review outputs, not canonical source.

## 5. Correlated model error

Multiple agents may repeat the same interpretation. Use independent prompts, role separation, source-first critics, and Sol adjudication; this reduces but cannot eliminate correlated error.

## 6. One-human ownership

The owner is not reviewing intermediate work. Autonomous gates must therefore be unusually explicit, and final residual uncertainty must be visible.

## 7. Synthetic data realism

Generated data may miss local naming conventions and historical ugliness. Include deliberately messy cases and clearly label it synthetic.

## 8. Visual quality without a human designer

LLMs can produce coherence but may converge on generic patterns. Require alternative directions, independent critique, contact sheets, distinctiveness testing, and a single integrated thesis.

## 9. Interconnected workflows

Upstream changes can affect many artifacts. “Frozen” means change-controlled, not immutable. Maintain impact analysis and traceability.

## 10. Static screens are insufficient

Specifications must include states, interactions, validation, permissions, offline behavior, failure handling, history, and data. This is mandatory in screen and construction packets.

## 11. Workflow does not equal software feature

Each workflow step requires a product-intervention decision. Some work remains external or is represented only through communication or records.

## 12. Capacity and interrupted runs

The project may span long execution and many subagent runs. Durable repository state, bounded tasks, checkpoints, and restart procedures are required.

## 13. GitHub capability drift

GitHub APIs and ruleset behavior may evolve. Verify current official docs when a command fails. Preserve intent: bot ownership, deterministic checks, PR integration, public Pages, and safe recovery.

## 14. Codex model/config drift

Model identifiers and project-scoped agent syntax may evolve. Verify current official OpenAI docs, use the newest Sol equivalent, and record any compatibility adjustment.

## 15. Figma plan dependence

MCP write access, Code Connect, and seats/plans may differ when future migration occurs. Completion requires readiness artifacts, not a live Figma file.
